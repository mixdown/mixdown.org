mixdown.js

Mixdown is a philosophy. Stop Hacking Middleware. Start Remixing. Loosely coupled plugins and modules are the core of node.js development. Mixdown is a set of libraries that help you develop any type of application (web, cli, etc) faster and cleaner. It can be consumed as a framework if you like or pick your components "dim-sum" style.

Remix over refactor

When you write something once, you should re-use it. Mixdown is a set of components that allow you to remix open source or your custom components.

Put a plugin on it (Plugins over middleware)

e.g. Plugins over middleware
Small discrete packages are a core concept of npm. Mixdown leverages flatiron/broadway for plugin interfaces.

You are the mix master

Plugins a discrete components. Mixdown lets you remix anything into your application. Quickly. While keeping your code clean and well-factored.

Distributed configuration (if you wanna)

Configuration based dependency injection allows storage of your site configuration anywhere. File system, zookeeper, any NoSql, or anywhere else you can think of. (Except carrier pigeon, but they aren't real time so you probably don't care)
